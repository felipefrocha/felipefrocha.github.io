# Internationalization (i18n) Guide

This project uses [react-i18next](https://react.i18next.com/) for internationalization, supporting **Portuguese**, **English**, and **Spanish** out of the box.

## Adding a New Language

Adding a new language is straightforward. Follow these steps:

### 1. Create Translation File

Create a new JSON file in `client/src/locales/` with the language code (e.g., `fr.json` for French, `de.json` for German):

```json
{
  "common": {
    "loading": "...",
    "error": "...",
    // ... copy structure from en.json
  },
  "nav": {
    "home": "...",
    // ... etc
  }
}
```

### 2. Update i18n Configuration

Edit `client/src/lib/i18n.ts`:

1. **Add the language code to `SUPPORTED_LANGUAGES`**:
```typescript
export const SUPPORTED_LANGUAGES = ['en', 'pt', 'es', 'fr'] as const;
```

2. **Add language metadata to `LANGUAGE_METADATA`**:
```typescript
export const LANGUAGE_METADATA: Record<SupportedLanguage, { name: string; nativeName: string; flag: string }> = {
  en: { name: 'English', nativeName: 'English', flag: '🇺🇸' },
  pt: { name: 'Portuguese', nativeName: 'Português', flag: '🇧🇷' },
  es: { name: 'Spanish', nativeName: 'Español', flag: '🇪🇸' },
  fr: { name: 'French', nativeName: 'Français', flag: '🇫🇷' }, // Add this
};
```

3. **Import and register the translation file**:
```typescript
import frTranslations from '../locales/fr.json';

// In the resources object:
resources: {
  en: { translation: enTranslations },
  pt: { translation: ptTranslations },
  es: { translation: esTranslations },
  fr: { translation: frTranslations }, // Add this
},
```

### 3. Translate All Keys

Make sure to translate **all** keys from `en.json` to maintain consistency. The structure should match exactly.

## Using Translations in Components

### Basic Usage

```typescript
import { useTranslation } from 'react-i18next';

function MyComponent() {
  const { t } = useTranslation();
  
  return <h1>{t('nav.home')}</h1>;
}
```

### With Interpolation

```typescript
const { t } = useTranslation();
<p>{t('blog.readTime', { minutes: 5 })}</p>
```

## Translation File Structure

The translation files are organized by feature/section:

- `common` - Shared strings (loading, error, etc.)
- `nav` - Navigation items
- `hero` - Hero section content
- `blog` - Blog-related strings
- `portfolio` - Portfolio page
- `about` - About page
- `contact` - Contact form
- `sections` - Section headers
- `work` - Work/project related strings

## Language Detection

The app automatically detects the user's language preference from:
1. **localStorage** (`i18nextLng` key) - User's manual selection
2. **Browser language** - Falls back to browser settings

The selected language is persisted in localStorage for future visits.

## Language Switcher

The language switcher is available in the header (top right). Users can click the globe icon to change languages.

## Best Practices

1. **Always use translation keys** - Never hardcode user-facing strings
2. **Keep keys descriptive** - Use namespaced keys like `blog.title` instead of `title`
3. **Maintain consistency** - All language files should have the same structure
4. **Test all languages** - Make sure UI doesn't break with longer/shorter translations
5. **Use interpolation** - For dynamic content, use i18next interpolation instead of string concatenation

## Example: Adding French

1. Create `client/src/locales/fr.json` (copy from `en.json` and translate)
2. Update `client/src/lib/i18n.ts`:
   - Add `'fr'` to `SUPPORTED_LANGUAGES`
   - Add French metadata to `LANGUAGE_METADATA`
   - Import and add `fr: { translation: frTranslations }` to resources
3. Done! The language will appear in the switcher automatically.
