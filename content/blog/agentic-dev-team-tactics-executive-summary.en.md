## The executive signal

Agentic development fails when it is treated as a tool rollout. The real change is operating-model design: who supervises agents, which gates are safe to flip, what context is required, and how the team knows whether autonomy is improving.

The first 90 days should not automate everything. They should build the map, select the first gates, observe agent divergence, and convert validated patterns into team standards.

## What must change

The team needs different paths for brownfield and greenfield work. Brownfield contexts usually start with regression-safe gates such as test generation and review support. Greenfield contexts can start with spec-to-scaffold workflows, but only when tickets include clear acceptance criteria, constraints, and expected output shape.

Each gate needs a supervisor, a definition of correct output, a shadow-mode period, and explicit flip criteria. This prevents agent use from becoming informal improvisation hidden inside individual developer workflows.

## The leadership takeaway

The practical target is not "everyone uses AI." The target is a repeatable gate pattern that can be handed from senior engineers to the broader team without losing quality.

By the end of 90 days, leadership should have baseline-to-current metrics, documented intervention reasons, a gate pattern, and a six-month roadmap for expanding autonomy where it is already proving reliable.
